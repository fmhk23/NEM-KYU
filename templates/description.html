Take A Holiday!

有給申請するページです。日付を選んでapply Holiday!を押すとモザイクを会社のアドレスに送信します。

会社のアドレスがモザイク受信すると、カレンダーに休みの予定が追加されます。

本来であれば認証などが必要でしょうが、テスト用のページとして、company_a社のAlice社員の有給申請用ページを用意しました。

テスト用ということで実環境とは異なるところが幾つかありますが、詳しく知りたい人は下の雑多な英語を読んで下さい。

Description

Basic Idea/Problems I want to solve

In Japan, many employees can't take all of their paid holidays.

https://www.travelvoice.jp/english/japan-is-the-worst-country-in-the-paid-holiday-acquisition-rate-ranking/

NEM-KYU is a simple application for Attendance Management.

With this application, employees can offer paid holiday by sending mosaics.

They don't have to make a phone-call/e-mail or talk with their boss.(I don't like all of them!)

Since it is working on public chain, anyone can visualize if the company is employee-friendly or not.

It is very helpful for job finders. They can know if they can take holidays easily or not.

Currently companys let us know the paid holiday axquisition rate, but can you trust it?

By using public chain, not employee-friendly company can't recruit anyone!

System Description

In this hackathon, I developed 2 functions.

Offering paid holiday by sending a mosaic.

In this system, company have its own NEM address and namespace.

It issues mosaics like "holiday2018" and distribute it to its employees.

When an employee feels he wants a holiday, he just needs to send a mosaic via NEM-KYU.

Instead of using NEM-KYU, he can send it using Nanowallet, too. (Need to set appropriate message, though.)

By triggering mosaic-recieve, company-side can run any program like sending messages to sender's colleagues, updating calendar and so on.

It means that it is very easy to integrate with current system.

In this test version, as company-side, transaction-monitoring program automatically calls program to update the Google calendar.

Not only updating calendar, but also any program can be called if developed!

For user-test, I prepare user "Alice" who is working for "company_a".

Select the date you want to take a holiday and click apply to send a mosaic.

After transaction is confirmed, check the google calendar is updated.

In test version, following features don't reflect the actual situation;

- Alice has tremendous holidays.

- Alice can offer holiday even if it is already offered/Sunday/long time ago.

- In actual system, it needs authentication but test version doesn't offer it.

- Destination address is preset.

- Can't use Nanowallet since I can't tell you Alice's private key.

Fixing them is easy but they are helpful to make many users test this app with few resources.

Paid Holiday Acquistion Dashboard

This dashboard helps you to check how many holidays are offered until now.

I prepared a sample company "company_b" for demo.

Company_b started in 2016 with 5 members.

Employees get 20 "company_b:holiday20xx" every year.(xx = 16,17,18)

Thus total supply of "company_b:holiday20xx" is 100.

Of course you can check company_a dashboard, too!

Since company_a has only one employee Alice and she has 1000 holidays, offered paid holiday ratio is very low.

Try NEM-KYU to increase the offered paid holiday ratio of company_a!

(test mosaic was used during development instead of holiday mosaic.)

Future Plan:

Update of NEM-KYU

With using blockchain, these functions would be easily implemented.

- Trading holidays with colleagues.

Even in current system, you can trade holidays with your colleagues using Nanowallet.

If company provides the exchange, you can trade your holidays more easily.

- Get XEM refunds for expired holidays.

I couldn't implement it in this hackathon since it seemed time-consuming to implement function to identify if the transaciton is holiday offer or refund offer.

However, it is a very simple idea and not difficult to implement:Just make a xem transaction when company-side receives holiday mosaic with "refund" message.

It is very nice idea to use multisig transaction when employee offers a holiday.

In Japan, company-side can ask employees to change the date of holiday.

With using multi-sig transaction, the boss can confirm the offer.

Additionally, it is possible to update Dashboard.

Current dashboard has only simple functions but it is possible to visualize when employees offers holidays.

Depending on their business, there are busy season and off-season. It can visualize them with actual data.

Other Field

Moreover, NEM-KYU is not just Attendance Management Application, but is useful for Emergency Contact.

At the time of disaster, company wants to know if their colleagues are safe or not as soon as possible.

In such cases, workers can tell by sending “Company_NAME:Safe” mosaic to company address.

Since it is completely same system with Attendance management, workers don’t need another training for using specific system.

If server is down due to disaster and can't access to NEM-KYU, users can tell by using nanowallet.

If their family/friends knows his address, they can know if he is safe by checking the transactions from the address.

No need to contact directly, which causes traffic jam in network.
